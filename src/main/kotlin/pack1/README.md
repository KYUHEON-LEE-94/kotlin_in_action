# 1장 코틀린이란 무엇인가

## 함수형 프로그래밍과 객체지향 프로그래밍
### 일급 시민인 함수
- 해당 요소(함수)를 변수처럼 "값"으로 다룰 수있다.
    - 1. 변수에 저장할 수 있다. 
  ```kotlin
    // 1. 함수를 값처럼 변수에 저장
    fun greet() = println("Hello")
    
    val sayHello = ::greet   // 함수 자체를 저장
    sayHello()               // 실행 -> Hello
  ```
    - 2. 함수의 인자로 넘길 수 있다
    ```kotlin
    // 2. 함수를 파라미터로 전달
    fun doSomething(action: () -> Unit) {
        action()
    }
    
    doSomething(::greet) // greet 함수를 전달하여 실행
  ```
    - 3. 함수가 함수를 반화할 수 있다 ex) return ::sum
    ```kotlin
      // 3. 함수를 반환값으로 사용
    fun getOperator(): (Int, Int) -> Int {
        return { a, b -> a + b }  // 람다를 반환
    }
    
    val plus = getOperator()
    println(plus(3, 5)) // 8
  ```
! 이 개념이 중요한 이유
- 코드를 데이터처럼 다룰 수 있게 되고
- 고차 함수(함수를 인자로 받거나 반환하는 함수)를 만들 수 있고
- 람다, map/filter/reduce 같은 함수형 스타일을 사용할 수 있게 된다.

### 불변성
함수형 프로그래밍에서는 일단 만들어지고 나면 내부 상태가 절대로 바꾸지 않는 불변 객체를 사용해 프로그램을 작성한다.

### 부수 효과 없음
함수형 프로그래밍에서는 입력이 같으면 항상 같은 출력을 내놓고 다른 객체의 상태를 변경하지 않으며, 
함수 외부나 다른 바깥 환경과 상호작용하지 않는 순수 함수를 사용해야한다.
